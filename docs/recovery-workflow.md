# Disk Recovery Workflow

This document explains the planned workflow for the Disk Recovery Image project.

## 1. Preparation

- Create a safe working folder for the investigation.
- Keep the original disk image unchanged.
- Work only on a copy of the image when possible.
- Record the file name, date, and purpose of the analysis.

## 2. Image Inspection

- Identify the disk image format.
- Check whether partitions or file systems are visible.
- Note basic information such as size, structure, and suspected file types.

## 3. Recovery Process

- Locate recoverable files.
- Recover selected files to a separate output folder.
- Avoid modifying the source image.
- Keep notes about which files were recovered and where they were found.

## 4. Documentation

For each recovery attempt, record:

- Tool or method used
- Date and time of recovery
- Files recovered
- Errors or damaged files
- Observations from metadata

## 5. Reporting

The final report should include:

- Objective of the recovery
- Disk image details
- Recovery steps
- Results
- Limitations
- Conclusion

## Learning Focus

This workflow helps practice digital forensics thinking: careful handling, repeatable steps, and clear evidence documentation.
